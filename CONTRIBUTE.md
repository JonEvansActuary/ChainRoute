# Contributing to ChainRoute Protocol

Thank you for your interest in contributing to ChainRoute! This is a specification-first project aimed at defining a lightweight provenance protocol. We welcome contributions that improve the docs, refine the spec, add examples, or propose extensions. All contributions are under the [MIT License](../LICENSE).

## Code of Conduct

This project adheres to the [Contributor Covenant Code of Conduct](../CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code. Please report unacceptable behavior to technogoogie@example.com.

## How to Contribute

### Reporting Bugs
- Use the GitHub Issues tab to check if the bug already exists.
- If not, open a new issue with:
  - A clear title and description.
  - Steps to reproduce.
  - Expected vs. actual behavior.
  - Screenshots or code snippets if relevant.
- Label it as "bug".

### Suggesting Enhancements
- Open an issue labeled "enhancement".
- Describe the problem it solves.
- Propose a solution (e.g., new field in JSON schema, multi-sig support).
- Reference related protocols or prior art.

### Pull Requests
1. Fork the repo and create your branch from `main` (e.g., `feature/new-example`).
2. Make your changes:
   - For spec/docs: Use clear markdown; update version if needed.
   - For code/examples: Add to `/examples/` or `/reference-impl/`; ensure it's MIT-licensed.
   - Keep commits atomic and descriptive.
3. Test your changes (e.g., validate JSON schemas).
4. Open a PR with:
   - A descriptive title.
   - Summary of changes.
   - Reference to related issues.
5. Sign off your commits (e.g., `git commit -s` for DCO agreement).

We aim to review PRs within a week. If it's a major change, discuss in an issue first.

## Style Guidelines
- **Markdown**: Use consistent headings, bullet lists, code blocks.
- **JSON/Code**: Indent with 2 spaces; use descriptive variable names.
- **Commit Messages**: Start with type (e.g., "docs: update spec for delegation") followed by a short description.

## Recognition
Contributors will be credited in the README or a CONTRIBUTORS file for significant work.
